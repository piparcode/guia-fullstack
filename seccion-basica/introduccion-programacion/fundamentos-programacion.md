## ¿Qué es la programación?

La programación es el proceso de crear instrucciones que una computadora puede seguir para llevar a cabo una tarea específica. Estas instrucciones se escriben en un lenguaje de programación y se traducen en un código que la computadora puede entender y ejecutar.

> No programes para solucionar problemas, programa para crear soluciones

### Ejemplo

Imagina que quieres darle instrucciones a un robot para que haga un sándwich. Para programar al robot, necesitas darle una serie de pasos claros y ordenados que pueda seguir. Aquí está el programa en lenguaje humano:

1. Toma dos rebanadas de pan.
2. Unta la mantequilla en una de las rebanadas.
3. Coloca una loncha de jamón en la rebanada con mantequilla.
4. Agrega una loncha de queso encima del jamón.
5. Pon la otra rebanada de pan encima para cerrar el sándwich.

Estos pasos le indican al robot exactamente qué debe hacer en cada etapa del proceso. Sin embargo, si le das estos pasos a un robot sin más información, no sabrá cómo realizar las acciones específicas.

Aquí es donde entra en juego la programación. Debes traducir esos pasos a un lenguaje que el robot pueda entender.

## Fundamentos de la programación 

Son los conceptos básicos y las ideas clave que debes entender para comenzar a programar de manera efectiva. Aquí hay algunos fundamentos importantes:

1. **Algoritmos:** Un algoritmo es una serie de pasos ordenados y precisos que se deben seguir para resolver un problema. Es la base de cualquier programa, ya que describe la lógica detrás de su funcionamiento.
    <br />
    - **Ejemplo de un algoritmo para contar hasta 10**
        - **Paso 1:** Comenzar en 1.
        - **Paso 2:** Decir en voz alta el número actual.
        - **Paso 3:** Aumentar el número en 1.
        - **Paso 4:** Si el número es menor o igual a 10, repetir desde el Paso 2.
        - **Paso 5:** Terminar.
<br />

2. **Estructuras de control:** Las estructuras de control te permiten controlar el flujo de ejecución de un programa. Las estructuras más comunes son las estructuras de selección (como las declaraciones "if" y "switch") y las estructuras de repetición (como los bucles "for" y "while").
    <br />
    - **Estructura de control "Si-Entonces"**
        - **Ejemplo:** Si hace sol, entonces vamos al parque. Si no, nos quedamos en casa.
    - **Estructura de control "Si-Entonces-Sino"**
        - **Ejemplo:** Si hace sol, entonces vamos al parque. Si no, nos quedamos en casa y jugamos dentro.
    - **Bucle "Mientras"**
        - **Ejemplo:** Mientras tengamos caramelos, los compartiremos con nuestros amigos.
    - **Bucle "Para"**
        - **Ejemplo:** Para cada amigo en la lista, les daremos una tarjeta de cumpleaños.
    <br />
    > Estas estructuras de control nos permiten tomar decisiones y repetir acciones en función de ciertas condiciones.

<br />

3. **Variables:** Las variables son espacios de memoria utilizados para almacenar datos. Cada variable tiene un nombre y un tipo de datos asociados. Puedes asignar valores a las variables y utilizarlos en tu programa para realizar cálculos o almacenar información.
    <br />
    - **Variable "Edad"**
        - **Ejemplo:** Supongamos que tu edad es 8. La variable "Edad" representa tu edad actual.
    - **Variable "Puntuación"**
        - **Ejemplo:** Durante un juego, puedes tener una variable llamada "Puntuación" que registra la cantidad de puntos que has obtenido.
    - **Variable "Nombre"**
        - **Ejemplo:** Si tienes una mascota, puedes tener una variable llamada "Nombre" que almacena el nombre de tu mascota.
    - **Variable "Cantidad de Dulces"**
        - **Ejemplo:** Imagina que tienes una bolsa de dulces. Puedes tener una variable llamada "Cantidad de Dulces" que indica cuántos dulces tienes en ese momento.
    <br />
    > Estos ejemplos simples de variables representan diferentes tipos de información que pueden cambiar o variar con el tiempo. Las variables permiten almacenar y manipular datos en programas y actividades.

4. **Tipos de datos:** Los tipos de datos definen los diferentes tipos de valores que pueden ser almacenados y manipulados en un programa. Algunos tipos de datos comunes son enteros (int), números de punto flotante (float), cadenas de texto (string) y booleanos (bool).
    <br />
    - **Tipo de dato "Número entero"**
        - **Ejemplo:** Puedes tener un número entero para representar la cantidad de lápices que tienes en tu estuche.
    - **Tipo de dato "Texto" o "Cadena"**
        - **Ejemplo:** Puedes tener un texto o cadena para representar tu nombre completo.
    - **Tipo de dato "Booleano"**
        - **Ejemplo:** Puedes tener un valor booleano para representar si estás o no usando zapatos en un momento dado (verdadero si los llevas puestos, falso si no).
    - **Tipo de dato "Lista" o "Arreglo"**
        - **Ejemplo:** Puedes tener una lista de nombres de amigos para mantener un registro de las personas con las que te gusta jugar.
    <br />
    > Estos ejemplos representan diferentes tipos de datos que se pueden utilizar en programas o actividades. Los números enteros representan valores numéricos, las cadenas representan texto, los valores booleanos representan opciones verdadero/falso, y las listas o arreglos permiten almacenar múltiples valores en una sola variable. Entender los diferentes tipos de datos es fundamental para trabajar con información en programación y otras áreas.

5. **Estructuras de datos:** Las estructuras de datos son formas de organizar y almacenar datos de manera eficiente. Algunas estructuras de datos comunes incluyen matrices (arrays), listas enlazadas (linked lists), pilas (stacks) y colas (queues).
    <br />
    - **Lista**
        - **Ejemplo:** Una lista de tus juguetes favoritos: ["Muñeca", "Pelota", "Rompecabezas"]
    - **Diccionario**
        - **Ejemplo:** Un diccionario de animales y sus sonidos: {"Perro": "Guau", "Gato": "Miau", "Vaca": "Muu"}
    - **Pila**
        - **Ejemplo:** Una pila de libros: [Libro3, Libro2, Libro1] (donde el último libro agregado se encuentra en la parte superior)
    - **Cola**
        - **Ejemplo:** Una cola de personas esperando su turno en un juego: [Persona1, Persona2, Persona3] (donde la persona en la posición 0 es la siguiente en jugar)
    <br />
    > Estos ejemplos ilustran diferentes estructuras de datos que se utilizan para organizar y almacenar información de manera específica. Una lista es una colección ordenada de elementos, un diccionario asocia claves con valores, una pila sigue el principio "último en entrar, primero en salir" (LIFO), y una cola sigue el principio "primero en entrar, primero en salir" (FIFO). Estas estructuras de datos son conceptos fundamentales en programación y ayudan a organizar y manipular datos de manera eficiente.

6. **Funciones:** Las funciones son bloques de código reutilizables que realizan una tarea específica. Puedes crear tus propias funciones para dividir un programa en partes más pequeñas y manejables, lo que facilita su mantenimiento y reutilización.
    <br />
    - **Función "Saludar"**
        - **Ejemplo:** Una función que imprime "¡Hola!" en la pantalla cuando se llama.
    - **Función "Sumar"**
        - **Ejemplo:** Una función que toma dos números como entrada y devuelve la suma de los mismos.
    - **Función "Doblar"**
        - **Ejemplo:** Una función que toma un número como entrada y devuelve el doble de ese número.
    - **Función "Mayor"**
        - **Ejemplo:** Una función que toma dos números como entrada y devuelve el número más grande de los dos.
    <br />
    > Estos ejemplos representan funciones que realizan acciones o cálculos específicos. Una función toma ciertos valores de entrada, realiza una serie de instrucciones y puede devolver un resultado. Las funciones son bloques de código reutilizables que permiten organizar y simplificar la lógica de un programa. Comprender cómo se pueden utilizar las funciones para realizar tareas específicas es un concepto importante en programación.

7. **Modularidad:** La modularidad es el principio de dividir un programa en módulos más pequeños y autónomos. Cada módulo debe tener una responsabilidad única y comunicarse con otros módulos a través de interfaces bien definidas. Esto promueve la legibilidad, el mantenimiento y la reutilización del código.
    <br />
    - **Juego de adivinanzas**
        - **Módulo 1:** Función para mostrar una adivinanza en la pantalla.
        - **Módulo 2:** Función para pedir al usuario una respuesta.
        - **Módulo 3:** Función para verificar si la respuesta del usuario es correcta.
    - **Construcción de una casa de bloques**
        - **Módulo 1:** Función para apilar bloques y construir la base de la casa.
        - **Módulo 2:** Función para colocar bloques en forma de paredes.
        - **Módulo 3:** Función para añadir un techo a la casa.
    <br />
    > Estos ejemplos ilustran cómo la modularidad permite dividir un programa o una tarea en partes más pequeñas y manejables. Cada módulo tiene una función específica y puede ser reutilizado en diferentes contextos. Esto facilita el desarrollo de programas más estructurados y fáciles de entender.

8. **Depuración:** La depuración es el proceso de encontrar y corregir errores en un programa. Es una habilidad esencial en programación y requiere el uso de técnicas y herramientas específicas para identificar y solucionar problemas.
    <br />
    - **Juego de adivinanzas**
        - **Problema:** El juego muestra respuestas incorrectas.
        - **Depuración:** Revisar la función que verifica las respuestas para asegurarse de que esté comparando correctamente la respuesta del usuario con la respuesta correcta.
    - **Programa de cálculo**
        - **Problema:** El programa muestra resultados incorrectos.
        - **Depuración:** Verificar las fórmulas y los cálculos realizados en el programa para encontrar posibles errores de lógica o errores matemáticos.
    - **Juego de laberinto**
        - **Problema:** El jugador no puede moverse correctamente por el laberinto.
        - **Depuración:** Revisar la lógica del movimiento del jugador y verificar si hay errores en la detección de paredes o en los controles del teclado.
    <br />
    > Estos ejemplos ilustran cómo la depuración implica identificar y crear soluciones para los problemas de un programa. Puede implicar revisar el código, buscar errores lógicos, rastrear el flujo del programa y realizar pruebas para encontrar y corregir errores. La depuración es una habilidad importante en programación y ayuda a desarrollar su capacidad de solucionar problemas y mejorar la calidad de sus programas.

<br />
